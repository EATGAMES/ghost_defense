# Merge Defense - 콘텐츠 구조

## 1. 캐릭터 구조

### 1.1 캐릭터 역할
이 게임에서 캐릭터는 두 가지 역할로 동작한다.

- 필드 캐릭터: 머지 대상이 되는 캐릭터
- 공격 캐릭터: 상단에서 보스를 공격하는 연출 캐릭터

단, 두 역할은 서로 다른 개체가 아니라 `단계에 따라 어떤 캐릭터가 등장하는지`라는 공통 규칙으로 연결된다.

### 1.2 5종 세팅
플레이어는 스테이지에 사용할 캐릭터 5종을 순서대로 세팅한다.

예시:
1. 방깨비
2. 구미오
3. 천여귀
4. 강시희
5. 도라큘

이 순서는 필드 등장 캐릭터와 상단 공격 캐릭터 모두에 직접 반영된다.

---

## 2. 단계와 캐릭터 매핑

### 2.1 반복 규칙
세팅한 5종은 단계에 따라 반복된다.

| 단계 | 캐릭터 |
|---:|---|
| 1단계 | 방깨비 |
| 2단계 | 구미오 |
| 3단계 | 천여귀 |
| 4단계 | 강시희 |
| 5단계 | 도라큘 |
| 6단계 | 방깨비 |
| 7단계 | 구미오 |
| 8단계 | 천여귀 |
| 9단계 | 강시희 |
| 10단계 | 도라큘 |

### 2.2 의미
- 단계가 먼저 결정된다.
- 그 단계에 대응하는 캐릭터가 자동으로 정해진다.
- 캐릭터는 단계 스킨이자 공격 연출 캐릭터 역할을 동시에 가진다.

---

## 3. 필드 캐릭터 규칙

### 3.1 필드 오브젝트 정체성
- 필드에는 공이 아니라 캐릭터가 등장한다.
- 하지만 머지 기준은 캐릭터 종류가 아니라 단계다.
- 따라서 필드 캐릭터는 `캐릭터처럼 보이는 단계 오브젝트`에 가깝다.

### 3.2 머지 규칙
- 같은 단계끼리만 머지된다.
- 캐릭터 종류가 달라도 단계가 같으면 머지된다.
- 머지 후에는 다음 단계 캐릭터가 생성된다.

예시:
- 1단계 방깨비 + 1단계 방깨비 = 2단계 구미오
- 2단계 구미오 + 2단계 구미오 = 3단계 천여귀
- 5단계 도라큘 + 5단계 도라큘 = 6단계 방깨비

### 3.3 비주얼 규칙
- 각 캐릭터는 10단계 전체에서 정확히 2번 등장한다.
- 따라서 캐릭터마다 단계별 10종 이미지를 만들 필요는 없다.
- 각 캐릭터는 필드용 이미지 2개만 가지면 된다.

예시:
- 방깨비: 1단계용, 6단계용
- 구미오: 2단계용, 7단계용
- 천여귀: 3단계용, 8단계용

---

## 4. 공격 캐릭터 규칙

### 4.1 상단 공격 캐릭터
- 상단에는 항상 현재 공격 캐릭터 1명만 보인다.
- 전투 시작 시 1번 캐릭터가 먼저 서 있다.
- 이후 머지가 발생할 때마다 생성된 단계에 맞는 캐릭터로 즉시 교체된다.

### 4.2 공격 발동
- 머지로 새로운 단계가 생성되면 해당 단계의 캐릭터가 공격한다.
- 공격은 몬스터 1회 타격이다.
- 공격 후 상단 캐릭터는 사라지지 않고 그대로 대기한다.

### 4.3 큐 처리
- 머지가 연속으로 발생하면 공격 정보를 큐에 쌓는다.
- 상단 캐릭터는 큐를 순서대로 처리하면서 계속 교체된다.
- 큐가 비면 마지막 캐릭터 상태를 유지한다.

---

## 5. 캐릭터 전투 스탯

### 5.1 기본 스탯
캐릭터는 다음과 같은 전투 수치를 가진다.

- 기본 공격력
- 단계 계수 보정
- 치명타 확률
- 치명타 배율
- 고단계 보너스
- 공격 연출 속도

### 5.2 데미지 기준
- 최종 데미지는 `생성된 단계`와 `해당 캐릭터 스탯`을 기준으로 계산한다.
- 같은 단계라도 어떤 캐릭터가 배정되는지에 따라 실제 성능이 달라질 수 있다.

---

## 6. 캐릭터 공격 속성

각 캐릭터는 아래 2가지 속성을 가진다.

### 6.1 데미지 타입
- 물리 데미지
- 마법 데미지
- 폭발 데미지

### 6.2 공격 방식
- 원거리 공격
- 근거리 공격
- 소환수 공격

예시:
- 방깨비: 물리 / 근거리
- 구미오: 마법 / 원거리
- 강시희: 폭발 / 근거리
- 유우령: 마법 / 소환수

### 6.3 약점 매칭 의미
- 몬스터는 데미지 타입 약점과 공격 방식 약점을 가질 수 있다.
- 캐릭터 속성과 몬스터 약점이 맞으면 추가 데미지를 준다.
- 1개 일치면 1.25배, 2개 일치면 1.50배다.

## 7. 카드 구조

### 7.1 카드 등장 타이밍
- 카드 선택은 누적 공격 20회마다 1번 발생한다.
- 공격 큐 처리 기준으로 실제 공격 횟수를 누적한다.
- 팝업에는 카드 풀에서 무작위 카드 2장을 표시하고, 플레이어는 1장을 선택한다.

### 7.2 카드 레벨
- 카드는 S/A/B/C 등급을 사용하지 않는다.
- 카드 종류마다 데이터는 1장만 존재한다.
- 처음 선택한 카드는 1레벨이 된다.
- 이미 보유한 카드를 다시 선택하면 같은 카드의 레벨이 1 증가한다.
- 카드 효과는 `기본 능력값 + 레벨업당 추가 능력값 x (현재 레벨 - 1)`로 계산한다.
- 카드 이미지는 카드 데이터에 PNG 스프라이트로 연결한다.

### 7.3 카드 종류

| 카드명 | 효과 | 업그레이드 시 |
|---|---|---|
| 깔끔한 성격 | N단계 이하 캐릭터 즉시 제거 | 제거 단계가 2단계까지 증가 |
| 건망증 | N번 발사 동안 1단계 캐릭터 등장하지 않음 | 지속 발사 횟수 증가 |
| 최후의 보루 | 10단계 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 행운의 부적 | 크리티컬 확률 n% 증가 | 크리티컬 확률 증가 |
| 소환사 | 소환공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 타격가 | 근접공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 저격수 | 원거리공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 강력한 파워 | 물리공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 신비한 기운 | 마법공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 예술적 폭발 | 폭발공격 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 기습 | 다음 공격 1회 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 상향 평준화 | 모든 데미지 n% 증가 | 데미지 증가 퍼센트 증가 |
| 파워샷 | N회 동안 발사 파워가 강력해짐 | 지속 발사 횟수 증가 |
| 랩다이아몬드 | 승리 시 다이아 n개 추가 | 추가 다이아 개수 증가 |
| 골드러쉬 | 승리 시 골드 n% 증가 | 골드 증가 퍼센트 증가 |
| 새발의 피 | 공격 시 n회 동안 한 단계 낮은 캐릭터 추가 공격 | 지속 횟수 증가 |
| 지우개 | 이번 발사에 부딪히는 n번째 캐릭터까지 제거 | 제거 가능 횟수 증가 |
| 예지몽 | 다음 발사 캐릭터를 n번 미리 봄 | 미리보기 횟수 증가 |
| 핌 입자 | N번 동안 크기를 작게 줄여서 발사 | 지속 발사 횟수 증가 |
| 위기 탈출 | 제일 아래쪽에 있는 캐릭터를 n개 제거 | 제거 개수 증가 |

## 8. UI 콘텐츠 기준

### 8.1 플레이어가 꼭 알아야 할 정보
플레이어는 다음 정보를 빠르게 이해할 수 있어야 한다.

- 현재 단계 캐릭터가 누구인지
- 다음 머지 결과가 어떤 캐릭터인지
- 상단에서 지금 누가 공격 중인지
- 몬스터 약점이 무엇인지
- 카드가 언제 뜨는지

### 8.2 단계 순환 표시
하단 또는 별도 UI에 단계 순환 구조를 보여준다.

예시:
방깨비 → 구미오 → 천여귀 → 강시희 → 도라큘 → 방깨비 → ...

---

## 9. 최종 요약

> 캐릭터 5종을 순서대로 세팅하면, 그 순서가 1단계부터 10단계까지 반복 적용된다. 필드에서는 단계 기준으로 캐릭터를 머지하고, 상단에서는 생성된 단계에 해당하는 캐릭터가 즉시 교체되어 보스를 공격한다. 여기에 캐릭터별 데미지 타입과 공격 방식, 몬스터 약점 시스템이 더해져 빌드와 상성 선택의 의미가 생긴다.
