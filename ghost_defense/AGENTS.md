## 환경

* Unity 6.4 (6000.4.4f1) 기준으로 답변한다
* 모바일 세로형 게임 프로젝트다(900x1960 기기에 세로로 맞추고 좌우는 기기마다 다른 여백으로 남김)

## 코드 작성 규칙

* 코드 주석은 한국어로 작성한다
* 스크립트(.cs) 파일만 수정하는 작업은 MCP를 사용하지 않는다
* Unity Project 창에서 직접 할 수 있는 작업(파일 이동, 이름 변경 등)은 MCP 대신 사용자에게 직접 하도록 안내한다
* Inspector에 노출되는 SerializeField 필드에는 반드시 `[Tooltip("한글 설명")]`을 추가한다

## 작업 진행 순서

지시사항을 받으면 아래 순서를 반드시 따른다:

1. **파일 안내**: 추가/수정할 파일 이름을 먼저 알려준다
2. **방법 설명**: 사용자가 직접 할 경우 어떻게 하는지 2줄 이로 간략히 설명한다
3. **선택 요청**: 기본적으로 되묻지 않고 바로 실행한다. 단, 사용자가 명시적으로 선택을 요청했거나 되물음을 요구한 경우에만 "직접 하시겠습니까, 제가 할까요?"를 묻는다
4. **작업 진행**: 위 기준에 따라 즉시 진행한다

## 연속 작업 규칙

작업을 한번에 3개 이상 진행하는 경우 TASK 목록을 만들어 보여주면서 진행한다

## 프로젝트 네이밍 가이드

### 1. 기본 규약

* 숫자는 3자리 형식 `001`, `002` ...

---

### 2. 폴더 구조

| 폴더            | 용도                          |
| :-------------- | :---------------------------- |
| `_Data/`      | ScriptableObject 데이터 파일  |
| `_Fonts/`     | 폰트 리소스                   |
| `_Materials/` | 공용 머티리얼                 |
| `_Prefabs/`   | 프리팹 (씬별 서브폴더 분류)   |
| `_Scenes/`    | 씬 파일                       |
| `_Scripts/`   | 스크립트 (씬별 서브폴더 분류) |
| `_Spine/`     | Spine 2D 애니메이션 에셋      |
| `_Sprites/`   | 2D 스프라이트 이미지          |

---

### 3. 리소스 스키마

| 분류                       | 접두사 (Prefix)     |
| :------------------------- | :------------------ |
| **프리팹**           | `PFB_`            |
| **오브젝트**         | `OBJ_`            |
| **이펙트**           | `VFX_`            |
| **텍스처**           | `TEX_`            |
| **머티리얼**         | `MAT_`            |
| **사운드**           | `SFX_` / `BGM_` |
| **이미지**           | `IMG_`            |
| **폰트**             | `FNT_`            |
| **텍스트**           | `TXT_`            |
| **ScriptableObject** | `SO_`             |
| **씬**               | `SCN_`            |
