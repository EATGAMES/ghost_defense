# Merge Defense - System & UI

## 1. Lobby Structure

### 1.1 Main Tabs
로비는 다음 기능을 중심으로 구성한다.

1. 강화: 캐릭터 성장과 스테이지용 5종 순서 세팅
2. 카드: 보유 카드 확인과 강화
3. 배틀: 현재 스테이지 정보 확인과 전투 시작
4. 상점: 재화 및 패키지 구매
5. 미션: 일일/업적 미션과 보상 수령

### 1.2 Character Order Setup UI
- 플레이어는 스테이지에 사용할 캐릭터 5종을 순서대로 세팅한다.
- 1번부터 5번까지 슬롯이 고정으로 보인다.
- 각 슬롯에는 캐릭터 초상과 순번이 함께 표시된다.
- 이 순서는 전투 중 단계 순환 규칙의 기준이 된다.

## 2. Battle UI Overview

### 2.1 Top Area
- 상단 좌측: 몬스터
- 상단 우측: 현재 공격 캐릭터
- 현재 공격 캐릭터는 머지 결과에 따라 즉시 교체된다.
- 공격 후에도 마지막 상태로 계속 서 있다.

표시 요소:
- 보스 체력바
- 보스 페이즈 정보
- 현재 공격 캐릭터 스프라이트
- 현재 공격 캐릭터 이름 또는 강조 효과

### 2.2 Middle Area
- 필드에는 캐릭터가 등장한다.
- 머지 기준은 단계 기준이다.
- 각 캐릭터는 단계에 따라 대응 스킨으로 보인다.
- 연속 머지가 발생하면 필드에서는 머지 연출이 이어지고, 전투는 공격 큐로 처리된다.

표시 요소:
- 단계별 캐릭터 머지 필드
- 머지 이펙트
- 데드라인
- 필드 경고 연출

### 2.3 Bottom Area
- 현재 발사 대기 캐릭터
- 다음 발사 예정 캐릭터
- 드래그 조준선
- 카드 게이지
- 메뉴 버튼

## 3. Grade Rotation Display

### 3.1 Purpose
플레이어가 `현재 단계가 어떤 캐릭터인지` 바로 이해할 수 있어야 한다.

### 3.2 Display Rule
하단 또는 별도 UI에 단계 순환 구조를 보여준다.

예시:
방깨비 → 구미오 → 천여귀 → 강시희 → 도라큘 → 방깨비 → ...

### 3.3 Recommended Information
- 1~10단계 전체 미리보기
- 현재 필드에 많이 남아 있는 핵심 단계 강조
- 다음 머지 결과 단계 표시

---

## 4. Top Attack Character Presentation

### 4.1 Initial State
- 스테이지 시작 시 1번 캐릭터가 몬스터 우측에 서 있다.

### 4.2 Single Merge State
- 머지 발생 시 생성된 단계 캐릭터를 확인한다.
- 상단 공격 캐릭터를 즉시 교체한다.
- 교체된 캐릭터가 1회 공격 모션을 재생한다.
- 공격 후에도 사라지지 않고 그대로 대기한다.

### 4.3 Continuous Merge State
- 짧은 시간 안에 머지가 여러 번 일어나면 모두 공격 큐에 저장한다.
- 상단 공격 캐릭터는 큐 순서대로 하나씩 교체된다.
- 마지막 공격이 끝나면 마지막 캐릭터 상태로 유지된다.

## 5. Card Popup UI

### 5.1 Trigger
- 누적 공격 20회마다 카드 선택 팝업 등장

### 5.2 Flow
1. 공격 20회 누적
2. 전투 일시 정지
3. 카드 2장 표시
4. 1장 선택
5. 전투 재개

### 5.3 Popup Composition
- 어두운 배경 DIM
- 카드 2장
- 카드 이름
- 카드 효과 요약

## 6. Upgrade UI

### 6.1 Character Upgrade
- 캐릭터 기본 공격력 강화
- 치명타 관련 강화
- 단계 보너스 강화
- 특수 효과 강화

### 6.2 Order Tuning
- 로비에서 5종 순서를 쉽게 바꿀 수 있어야 한다.
- 순서 변경 결과가 전투에서 어떤 단계에 배정되는지도 함께 보여주는 것이 좋다.

---

## 7. Visual Identity Rule

### 7.1 Field Character Visual
- 필드 캐릭터는 단계별 순환 구조에 따라 외형이 정해진다.
- 캐릭터당 모든 단계 디자인을 만드는 구조는 아니다.
- 각 캐릭터는 2개의 단계용 이미지 세트를 가진다.

### 7.2 Top Character Visual
- 상단 공격 캐릭터는 필드 캐릭터보다 더 크게, 더 또렷하게 보이게 한다.
- 플레이어가 `지금 누가 공격 중인지` 즉시 알 수 있어야 한다.

### 7.3 Separation Rule
- 필드 캐릭터와 상단 공격 캐릭터는 같은 캐릭터라도 역할이 다르다.
- 따라서 크기, 위치, 강조 연출로 역할 차이를 분명하게 보여줘야 한다.

## 8. Development Priority Suggestion

### 8.1 Phase 1
- 단계 기준 머지
- 단계별 캐릭터 순환 스킨 적용
- 보스 체력 감소

### 8.2 Phase 2
- 상단 단일 공격 캐릭터 교체 연출
- 공격 큐 처리
- 마지막 공격 캐릭터 대기 구조

### 8.3 Phase 3
- 20공격 카드 팝업
- 단계 순환 UI 표시
- 로비 순서 세팅 UI 연결

---

## 9. Final Summary

> UI의 핵심은 플레이어가 `어떤 캐릭터 5종을 어떤 순서로 세팅했는지`, `지금 필드 단계가 어떤 캐릭터인지`, `상단에서 현재 누가 공격 중인지`를 한눈에 이해하게 만드는 것이다. 필드는 단계 순환 스킨 구조, 상단은 단일 공격 캐릭터 교체 구조를 기준으로 설계한다.
