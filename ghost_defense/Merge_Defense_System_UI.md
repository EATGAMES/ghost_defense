# Merge Defense - System & UI

## 2. 로비 화면 구성

### 2.1 로비 UI 구조 (메인 탭)
게임의 메인 화면은 하단 5개 탭으로 구성된다.

1. **강화**: 캐릭터 스탯 강화 및 5인 로스터 순서 세팅
2. **카드**: 보유 카드 확인 및 강화
3. **배틀**: 스테이지 정보 확인 및 게임 시작
4. **상점**: 재화 및 패키지 구매
5. **미션**: 일일/업적 미션 및 보상 수령

## 17. 영구 업그레이드 시스템 (강화 탭)

### 17.1 캐릭터 강화
- 캐릭터 기본 공격력 강화
- 크리티컬 확률 강화
- 크리티컬 데미지 강화
- 연타/광역/특수효과 계수 강화

### 17.2 로스터 순서 세팅
- 로비에서 5종 캐릭터를 장착한다
- 장착 순서는 배틀 중 실제 공격 순서에 직접 반영된다
- 왼쪽 1번이 첫 공격자다

## 23. UI 상세 구성

### 23.0 해상도 및 화면 규격
- 기준 해상도: **900 x 1960 (세로)**
- 좌우 여백은 BG만 표시
- 실제 플레이 영역은 900x1960 내부 고정

### 23.1 강화 탭 (Upgrade)
- 상단에 **로스터 슬롯 5개** 고정 배치
- 각 슬롯에는 장착 캐릭터 초상화와 순번 표시
- 드래그 또는 버튼으로 순서 변경 가능
- 캐릭터 상세 팝업에서 스탯 강화 버튼 노출

### 23.2 카드 탭 (Card)
- 카드 전체 목록 그리드
- 카드 레벨, 효과, 강화 버튼 표시

### 23.3 배틀 진입 화면
- 현재 장착한 5종 캐릭터와 순서 미리보기
- 추천 스테이지 정보
- 시작 버튼

### 23.5 인게임 UI

#### 상단 1/5
- 보스 체력바
- 보스 페이즈 아이콘
- **5인 공격 대기열 UI**
- 현재 공격 차례 캐릭터 강조
- 공격 큐 처리 연출

#### 중단 3/5
- 단계 오브젝트 머지 필드
- 필드 오브젝트는 캐릭터 랜덤 스킨으로 보이지만, 실제 판정은 단계 기준
- 데드라인 표시
- 연속 머지 이펙트

#### 하단 1/5
- 현재 발사 오브젝트 확대 표시
- 다음 생성 오브젝트 표시
- 조준선 표시
- **20공격 카드 게이지**
- 메뉴 버튼

### 23.6 인게임 카드 선택 팝업
- 누적 공격 20회 도달 시 노출
- 카드 2장 중 1장 선택
- 리롤 버튼 제공
- 선택 전까지 전투 일시 정지

## 24. 아트 스타일

### 24.1 캐릭터 표현
- 상단 공격 대기열 캐릭터는 개성이 분명하게 보이도록 크게 표현
- 필드 오브젝트는 캐릭터 스킨을 공유하지만 더 단순화된 축소형 표현 사용

### 24.2 머지 시각 변화
- 단계가 올라갈수록 크기와 장식 강화
- 10단계 완성 시 강한 소멸 연출

---

## 26. 밸런스 체크 포인트

### 26.1 공격 대기열 회전
- 특정 캐릭터가 너무 자주 오거나 너무 늦게 오는지 점검 필요
- 로스터 순서 세팅이 전략이 되도록 유지

### 26.2 20공격 카드 타이밍
- 카드가 너무 자주 뜨면 흐름이 끊김
- 너무 늦게 뜨면 빌드 재미가 약해짐

### 26.3 필드 스킨 혼동 방지
- 필드 오브젝트와 상단 공격 캐릭터가 같은 존재로 오해되지 않도록 UI 구분 필요

## 27. 개발 우선순위 제안

### 27.1 1차 프로토타입
- 단계 오브젝트 발사
- 같은 단계 머지
- 데드라인 패배 처리

### 27.2 2차 프로토타입
- 5인 공격 대기열
- 머지 단계 기반 공격 큐
- 공격 후 좌측 순번 순환

### 27.3 3차 프로토타입
- 20공격 카드 팝업
- 카드 효과 실제 적용
- 필드 랜덤 캐릭터 스킨 적용

### 27.4 4차 프로토타입
- 로비 순서 세팅
- 영구 강화
- 광고/VIP/경제 시스템
