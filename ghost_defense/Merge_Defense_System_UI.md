# Merge Defense - 시스템 UI

## 1. 로비 구조

### 1.1 메인 탭
로비는 다음 기능을 중심으로 구성한다.

1. 강화: 캐릭터 성장과 스테이지용 5종 순서 세팅
2. 카드: 보유 카드 확인과 강화
3. 배틀: 현재 스테이지 정보 확인과 전투 시작
4. 상점: 재화 및 패키지 구매
5. 미션: 일일/업적 미션과 보상 수령

### 1.2 캐릭터 순서 세팅 UI
- 플레이어는 스테이지에 사용할 캐릭터 5종을 순서대로 세팅한다.
- 1번부터 5번까지 슬롯이 고정으로 보인다.
- 각 슬롯에는 캐릭터 초상과 순번을 표시한다.
- 이 순서는 전투 중 단계 순환 규칙의 기준이 된다.

---

## 2. 전투 UI 개요

### 2.1 상단 영역
- 상단 좌측: 보스
- 상단 우측: 현재 공격 캐릭터
- 현재 공격 캐릭터는 머지 결과에 따라 즉시 교체된다.
- 공격 후에는 마지막 상태로 계속 서 있다.

표시 요소:
- 보스 체력바
- 보스 스테이지 정보
- 몬스터 약점 아이콘 또는 텍스트 0~2개
- 현재 공격 캐릭터 스프라이트
- 현재 공격 캐릭터 이름 또는 강조 효과

### 2.2 중단 영역
- 필드에는 캐릭터가 등장한다.
- 머지 기준은 단계 기준이다.
- 각 캐릭터는 단계에 따라 다른 필드 스킨으로 보인다.
- 연속 머지가 발생하면 필드에서는 머지 연출이 이어지고, 전투는 공격 큐로 처리된다.

표시 요소:
- 단계별 캐릭터 머지 필드
- 머지 이펙트
- 데드라인
- 필드 경고 연출

### 2.3 하단 영역
- 현재 발사 대기 캐릭터
- 다음 생성 예정 캐릭터
- 드래그 조준선
- 카드 게이지
- 메뉴 버튼

## 3. 단계 순환 표시 UI

### 3.1 목적
플레이어가 `현재 단계가 어떤 캐릭터인지`를 빠르게 이해할 수 있어야 한다.

### 3.2 표시 규칙
하단 또는 별도 UI에 단계 순환 구조를 보여준다.

예시:
방깨비 → 구미오 → 천여귀 → 강시희 → 도라큘 → 방깨비 → ...

### 3.3 추천 정보
- 1~10단계 전체 미리보기
- 현재 필드에 많이 남아 있는 핵심 단계 강조
- 다음 머지 결과 단계 표시

### 3.4 속성 힌트 표시
- 각 단계 캐릭터가 어떤 데미지 타입과 공격 방식을 가지는지 간단한 아이콘으로 보여줄 수 있다.
- 플레이어가 몬스터 약점에 맞는 단계를 빠르게 판단할 수 있어야 한다.

---

## 4. 상단 공격 캐릭터 연출 UI

### 4.1 초기 상태
- 스테이지 시작 시 1번 캐릭터가 몬스터 우측에 서 있다.

### 4.2 단일 머지 상태
- 머지 발생 시 생성된 단계 캐릭터를 확인한다.
- 상단 공격 캐릭터를 즉시 교체한다.
- 교체된 캐릭터가 1회 공격 모션을 재생한다.
- 공격 후 사라지지 않고 그대로 대기한다.

### 4.3 연속 머지 상태
- 짧은 시간 안에 여러 머지가 발생하면 모두 공격 큐에 저장한다.
- 상단 공격 캐릭터는 큐 순서대로 하나씩 교체된다.
- 마지막 공격이 끝나면 마지막 캐릭터 상태를 유지한다.

### 4.4 약점 피드백
- 약점 1개 일치 시 상단 공격 캐릭터나 데미지 숫자에 1단계 강조 연출
- 약점 2개 일치 시 더 강한 강조 연출
- 플레이어가 약점 보너스가 적용됐다는 사실을 즉시 인지할 수 있어야 한다

---

## 5. 보스 약점 표시 UI

### 5.1 표시 대상
- 데미지 타입 약점
- 공격 방식 약점

### 5.2 표시 방식
- 보스 체력바 근처에 아이콘 0~2개 표시
- 또는 짧은 텍스트 태그로 표시

예시:
- 폭발
- 근거리
- 마법 / 원거리

### 5.3 표시 목적
- 플레이어가 현재 어떤 단계 캐릭터를 만들면 유리한지 즉시 판단하게 한다.
- 약점 시스템이 숨겨진 수치가 아니라 명확한 전략 정보로 보이게 한다.

---

## 6. 카드 팝업 UI

### 6.1 발생 조건
- 누적 공격 20회마다 카드 선택 팝업 등장
- 카드에는 등급이 없고, 같은 카드를 다시 선택하면 레벨이 오른다.

### 6.2 흐름
1. 공격 20회 누적
2. 전투 일시 정지
3. 카드 2장 표시
4. 1장 선택
5. 전투 재개

### 6.3 팝업 구성
- 어두운 배경 DIM
- 카드 2장
- 카드 이미지
- 카드 이름
- 현재 레벨과 선택 후 레벨
- 카드 효과 요약
- 현재 레벨 기준 능력값
- 레벨업당 추가 능력값 또는 다음 레벨 증가량

### 6.4 카드 표시 규칙
- 카드 등급 텍스트는 표시하지 않는다.
- 처음 등장한 카드는 `Lv.1`로 표시한다.
- 이미 보유한 카드는 `Lv.현재 → Lv.다음` 형태로 표시한다.
- 카드 효과 수치는 `기본 능력값 + 레벨업당 추가 능력값 x (레벨 - 1)` 기준으로 계산해 보여준다.
- 카드 데이터에 연결된 PNG 스프라이트가 있으면 카드 이미지 영역에 표시한다.

## 7. 강화 UI

### 7.1 캐릭터 강화
- 캐릭터 기본 공격력 강화
- 치명타 관련 강화
- 단계 보너스 강화
- 특수 효과 강화

### 7.2 순서 조정
- 로비에서 5종 순서를 자유롭게 바꿀 수 있어야 한다.
- 순서 변경 결과가 전투에서 어떤 단계에 배정되는지 미리 보여주는 것이 좋다.

---

## 8. 비주얼 정체성 규칙

### 8.1 필드 캐릭터 비주얼
- 필드 캐릭터는 단계 순환 구조에 따라 외형이 정해진다.
- 캐릭터당 모든 단계 디자인을 만드는 구조가 아니다.
- 각 캐릭터는 2개의 필드 이미지 세트를 가진다.

### 8.2 상단 캐릭터 비주얼
- 상단 공격 캐릭터는 필드 캐릭터보다 더 크게, 더 선명하게 보여야 한다.
- 플레이어가 `지금 누가 공격 중인지`를 즉시 인지할 수 있어야 한다.

### 8.3 역할 분리
- 필드 캐릭터와 상단 공격 캐릭터는 같은 캐릭터라도 역할이 다르다.
- 따라서 크기, 위치, 강조 연출로 역할 차이를 분명히 보여줘야 한다.

## 9. 개발 우선순위 제안

### 9.1 1단계
- 단계 기준 머지
- 단계별 캐릭터 순환 스킨 적용
- 보스 체력 감소

### 9.2 2단계
- 상단 단일 공격 캐릭터 교체 연출
- 공격 큐 처리
- 마지막 공격 캐릭터 대기 구조

### 9.3 3단계
- 보스 약점 표시 UI
- 약점 일치 강조 연출
- 20공격 카드 팝업
- 단계 순환 UI 표시

---

## 10. 최종 요약

> UI의 목적은 플레이어가 `어떤 캐릭터 5종을 어떤 순서로 세팅했는지`, `지금 필드 단계가 어떤 캐릭터인지`, `상단에서 지금 누가 공격 중인지`, `보스 약점이 무엇인지`를 빠르게 이해하게 만드는 것이다. 필드는 단계 순환 스킨 구조, 상단은 단일 공격 캐릭터 교체 구조, 보스는 약점 표시 구조를 기준으로 설계한다.
